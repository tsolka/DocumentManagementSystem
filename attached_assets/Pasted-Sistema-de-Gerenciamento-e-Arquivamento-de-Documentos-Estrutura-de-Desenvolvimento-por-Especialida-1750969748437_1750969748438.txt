Sistema de Gerenciamento e Arquivamento de Documentos - Estrutura de Desenvolvimento por Especialidades
1. Arquiteto de Sistema
Prompt:
"Você é um arquiteto de sistemas sênior especializado em soluções de gerenciamento documental. Projete a arquitetura completa de um sistema de arquivamento de documentos com os seguintes requisitos:

Requisitos Funcionais:

Suporte para múltiplos formatos (PDF, JPEG, PNG, documentos Office)
Interface web para upload e cadastro com metadados
Processamento OCR assíncrono
Sistema de busca com indexação otimizada
Visualização in-browser de documentos
Download de arquivos originais
Requisitos Não-Funcionais:

Alta performance para grandes volumes
Escalabilidade horizontal
Segurança e controle de acesso
Backup e recuperação
Entregáveis:

Diagrama de arquitetura completo (componentes e fluxos)
Stack tecnológico recomendado com justificativas
Padrões de design e melhores práticas
Estratégia de microserviços vs monolito
Plano de escalabilidade e performance"
2. Desenvolvedor Frontend/UX
Prompt:
"Você é um desenvolvedor frontend sênior e especialista em UX. Desenvolva a interface de usuário para um sistema de gerenciamento documental com duas telas principais:

Tela 1 - Upload e Cadastro:

Design responsivo e intuitivo
Drag-and-drop para múltiplos arquivos
Formulário com campos: título, descrição, assunto, tags, data, categoria, departamento
Preview de arquivos antes do upload
Indicadores de progresso
Validações client-side
Tela 2 - Busca e Visualização:

Interface de busca avançada com filtros
Resultados em grid/lista com paginação
Preview de documentos inline
Visualizador de documentos integrado
Botões de ação (download, editar metadados, compartilhar)
Entregáveis:

Wireframes e mockups
Código frontend (React/Vue/Angular)
Sistema de componentes reutilizáveis
Testes de interface
Documentação de componentes"

4. Especialista em Banco de Dados
Prompt:
"Você é um DBA especialista em sistemas de grande escala. Projete e implemente a estrutura de dados para um sistema de arquivamento documental:

Requisitos de Armazenamento:

Suporte para milhões de documentos
Armazenamento de arquivos binários (até 100MB cada)
Metadados estruturados e texto extraído
Indexação full-text para busca rápida
Backup incremental
Análise Necessária:

Comparação entre PostgreSQL, MongoDB, e Elasticsearch
Estratégia de particionamento
Índices otimizados para busca
Armazenamento de objetos (S3/MinIO)
Entregáveis:

Modelo de dados completo
Scripts DDL
Estratégia de indexação
Plano de backup e recovery
Documentação de otimizações"
